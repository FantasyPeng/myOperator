## 二、多进程题目 

### 2.1 mysys.c: 

实现函数mysys，用于执行一个系统命令，要求如下
-	mysys的功能与系统函数system相同，要求用进程管理相关系统调用自己实现一遍

-	使用fork/exec/wait系统调用实现mysys

-	不能通过调用系统函数system实现mysys

#### 解题思路：
开辟子进程，使用excel函数执行命令，实现system的功能。

### 2.2 sh1.c
实现shell程序，要求具备如下功能
-	支持命令参数
-	$ echo arg1 arg2 arg3
-	$ ls /bin /usr/bin /home
-	实现内置命令cd、pwd、exit
-	$ cd /bin
-	$ pwd
-	/bin

#### 解题思路：
设置一个循环输入模仿shell终端，读取用户输入，并进行字符串拆分，判断执行何种操作。

echo:字符串切割后识别出为echo,创建新进程，再将参数输出来，父进程等待子进程执行完毕再继续执行。

ls:创建新进程，在新进程中调用execlp执行，字符串分割得到ls的参数结果。

cd:创建新进程，在其中调用chdir函数。

pwd:直接使用system(“pwd”)实现。

exit:该函数返回-1,主函数接收到后退出程序。

### 2.3 sh2.c: 
实现shell程序，要求在第1版的基础上，添加如下功能
-	实现文件重定向
-	$ echo hello >log
-	$ cat log
-	hello

#### 解题思路：
判断为echo命令后，继续判断拆分的字符串数组中有无”>”出现，如果有，则把“>”前的字符串写入文件,文件名为”>”后的字符串。


### 2.4 sh3.c:
 实现shell程序，要求在第2版的基础上，添加如下功能
-	实现管道
-	$ cat /etc/passwd | wc -l
-	实现管道和文件重定向
-	$ cat input.txt
-	3
-	2
-	1
-	3
-	2
-	1
-	$ cat <input.txt | sort | uniq | cat >output.txt
-	$ cat output.txt
-	1
-	2
-	3

#### 解题思路：
首先进行字符串解析，将字符串以“|“为分隔符分隔成为字符串数组，再使用递归函数，开辟子进程，通过pipe创建管道，子进程向管道读写内容，使用exec函数加载具体的操作。


## 三、多线程题目
### 3.1 pi1.c: 使用2个线程根据莱布尼兹级数计算PI
-	莱布尼兹级数公式: 1 - 1/3 + 1/5 - 1/7 + 1/9 - ... = PI/4
-	主线程创建1个辅助线程
-	主线程计算级数的前半部分
-	辅助线程计算级数的后半部分
-	主线程等待辅助线程运行Y束后,将前半部分和后半部分相加

#### 解题思路：
输入要计算的项数，主线程计算级数的前半部分，主线程等待辅助线程运行Y束后,将前半部分和后半部分相加。

### 3.2 pi2.c: 使用N个线程根据莱布尼兹级数计算PI
-	与上一题类似，但本题更加通用化，能适应N个核心，需要使用线程参数来实现
-	主线程创建N个辅助线程
-	每个辅助线程计算一部分任务，并将结果返回
-	主线程等待N个辅助线程运行结束，将所有辅助线程的结果累加

#### 解题思路：
计算公式前LEN项，读取用户输入辅助线程数量N,使用pthread_create(&workers[thread_num -1],NULL,compute,&myparams[thread_num]);
创建N个线程，分别进行计算，再通过pthread_join(workers[j],(void **)&myresult);得到返回的结果。主程序对结果进行相加。

### 3.3 sort.c: 多线程排序
-	主线程创建一个辅助线程
-	主线程使用选择排序算法对数组的前半部分排序
-	辅助线程使用选择排序算法对数组的后半部分排序
-	主线程等待辅助线程运行Y束后,使用归并排序算法归并数组的前半部分和后半部分

#### 解题思路：
主程序排序数组的前半部分，辅助程序排序后半部分，排序使用选择排序，主程序再使用归并排序进行排序。主进程和辅助进程分别使用选择排序算法对数组的前半部分和后半部分排序，产生中间结果。归并算法对其处理并放入结果数组，扫描指针分别放到中间结果数组开头和中间。


### 3.4 pc1.c: 使用条件变量解决生产者、计算者、消费者问题
-	系统中有3个线程：生产者、计算者、消费者
-	系统中有2个容量为4的缓冲区：buffer1、buffer2
-	生产者生产'a'、'b'、'c'、‘d'、'e'、'f'、'g'、'h'八个字符，放入到buffer1
-	计算者从buffer1取出字符，将小写字符转换为大写字符，放入到buffer2
-	消费者从buffer2取出字符，将其打印到屏幕上

#### 解题思路：添加了计算者，那么就需要一个mutex1限制buffer1的使用，mutex12限制buffer2的使用，还需要条件变量判断buffer1空或者已满，buffer2空或者已满。


### 3.5 pc2.c: 使用信号量解决生产者、计算者、消费者问题
-	功能和前面的实验相同，使用信号量解决

#### 解题思路：
首先，使用结构体定义信号量。wait为锁函数，使用wait锁住后，value -1,signal为释放互斥量，使用signal释放后value+1。
生产者：wait buffer1空然后放入字符，然后signal buffer1满。 
计算者：wait buffer1满然后取字符变大写，然后wait buffer2空，然后放入字符，然后signal buffer2，signal buffer1。   
消费者：wait buffer2满然后取字符输出，然后signal buffer2空

### 3.6 ring.c: 创建N个线程，它们构成一个环
-	创建N个线程：T1、T2、T3、… TN
-	T1向T2发送整数1
-	T2收到后将整数加1
-	T2向T3发送整数2
-	T3收到后将整数加1
-	T3向T4发送整数3
-	…
-	TN收到后将整数加1
-	TN向T1发送整数N

#### 解题思路：
主线程创建一个子线程，该子线程执行一个add函数，add函数对整数加1，并且设置计数器Count,当Count小于N时，在add函数中再创建子进程，同时子进程中继续调用add函数。
